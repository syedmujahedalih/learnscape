# Bring Your Own World roadmap

Learnscape should let creators build structured learning worlds, then decide who can use them. It should not let public users publish arbitrary code or unvalidated simulations.

## World package

Each world is a versioned, schema-validated package with:

- source and attribution
- concepts, variables, equations, and assumptions
- the domain model and allowed interventions
- learner predictions, evidence checks, and transfer prompts
- validation status and known limitations

Provider credentials are never part of a world package.

## Sharing modes

1. **Private draft** — visible only to the creator and invited collaborators.
2. **Link-only** — read-only, shareable with a specific class or peer group.
3. **Public** — discoverable, versioned, attributable, and moderated.
4. **Remix** — a fork that keeps source and creator lineage visible.

## Smallest useful MVP

1. Save a world privately.
2. Publish a read-only link.
3. Let a new creator remix it with attribution.
4. Add instructor access to completion and misconception evidence.

Collaborative editing, a public marketplace, and creator payouts wait until the flagship world has repeat use.

## Business model

Students can explore public worlds for free. Instructors and departments pay for private worlds, assignments, learning evidence, shared libraries, and administrative controls. Contributors first earn attribution, remix lineage, and featured placement; revenue sharing should start only once paid world distribution exists.
