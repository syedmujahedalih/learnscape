Inference engineering is usually learned in production—when latency is already burning and every experiment is expensive.

P99 is a flight simulator for inference engineers.

Instead of another architecture lecture, it drops you into a launch-day incident. Traffic jumped six times. An eight-billion-parameter service is missing its latency objective, the queue is compounding, and revenue is at risk.

You have to protect five constraints at once: tail latency, throughput, memory, quality, and cost.

First, commit to a failure mode. Then change the serving stack: precision, continuous batching, K-V cache, concurrency, prefix caching, and speculative decoding.

P99 does not reveal a canned answer. A compact learned world model recursively rolls the system state forward—queue depth, active requests, V-RAM, utilization, throughput, and P-ninety-five latency.

Here, the baseline collapses. Token demand outruns decode capacity and the queue explodes.

Now I quantize to INT four, batch continuously, reuse the shared prefix, and enable speculative decoding. The model forecasts a stable system.

Then an independent reference trace grades that forecast. P-ninety-five falls below four seconds, throughput clears demand, memory stays safe, and the incident is contained.

The checked-in network has six hundred and thirty parameters and was trained on ten thousand eight hundred bootstrap transitions. P99 labels that provenance explicitly. It is a real learned next-state model, not a fake production benchmark.

The next layer is already implemented: an authenticated Modal runner can launch a short-lived T-four, L-four, or A-ten G, execute llama dot C-P-P, and capture real server and NVIDIA telemetry for retraining.

Codex with GPT five point six helped me pressure-test the original education idea, pivot to inference engineering, design the product, implement the learned rollout and cloud trace architecture, write the tests, and deploy the final experience.

P99 turns systems intuition into something you can practice: predict, intervene, validate, and understand why.
